# Шпаргалка

## Создание локального репозитория

- `mkdir <название>`
- `cd <название>`
- `touch README.md`
- `git init`

## Создание удаленного репозитория

- меню-profiles-repositories-new
- ввести название (удобно, если совпадает с локальным) и отметить, публичный он будет или приватный
- нажать create repository
- скопировать ссылку на репозиторий SSH

## Связать локальный репозиторий с удаленным

- `git remote add origin <ссылка на репозиторий>`
- `git remote -v` - чтоб убедиться, что репозитории связаны (`-v` коротно от `--verbose` - подробный)   

## Порядок добавления коммита

- `touch <название>` - сначала создается сам файл через команду 
- `git add` - позволяет подтоговить файлы к сохранению/отслеживанию
- `git commit -m ''` - делаем коммит. Флаг `-m` позволяет присвоить сообщение в кавычках
- `git push` - отправить изменения в удаленный репозиторий. 
В первый раз эту команду нужно вызвать с флагом `-u` и параметрами `origin` (имя удалённого репозитория) и `main` или `master` (название текущей ветки). Флаг `-u` свяжет локальную ветку с одноимённой удалённой: 
`git push -u origin master`

## Хэширование

**Хэширование** - способ преобразовать набор данных и получить их "отпечаток". 
Информация о коммите - набор данных: 
- когда был сделан коммит
- содержимое файлов на момент коммита 
- сслыка на предыдущий или родительский

**SHA-1** - алгоритм, с помощью которого Git хэширует (преобразует) информацию о коммите и получает для каждого коммита свой уникальный **хэш**.
Хэши можно посмотреть в **.git/log/HEAD** или по команде `git log`.
